## 重要差分 詳細（上位5件）

### 差分1: pm 補正なし（GAS が renderer より劣る）

| | renderer | GAS |
|---|---|---|
| 入力 | `3時半 MTG` | ← |
| startTime | `15:30`（pm 補正あり） | `03:30`（補正なし） |
| endTime | `16:30` | `null` |

**原因**: GAS は `午後/午前` 前置詞がない場合の pm 推定ロジック（`pmGuess`）を持たない。renderer は1〜7時を自動的に +12h する。  
**どちらが正しいか**: renderer が正しい（音声入力「3時」は午後3時を意図することが多い）。  
**SPEC の2026-05-06 バグ修正該当**: 該当なし（既設計差分）。

---

### 差分2: 来週/今週曜日の週算出ロジック差異

| | renderer | GAS |
|---|---|---|
| 入力 | `来週月曜 10時 朝礼` | ← |
| date | `2026-04-13`（来週月曜） | `2026-04-20`（来々週月曜） |
| date（今週月曜）| `2026-04-06`（過ぎた月曜） | `2026-04-13`（来週月曜） |
| date（水曜日）| `2026-04-08`（本日水曜） | `2026-04-15`（来週水曜） |

**原因**: 週の起算点が異なる。renderer は ISO 週（月曜=0）ベース。GAS は日曜=0の `getDay()` ベース。「来週」のオフセット計算式が違う。  
**どちらが正しいか**: renderer が正しい（`来週月曜 10時 朝礼` で基準日2026-04-08(水)の来週月曜は4/13）。  
**SPEC の2026-05-06 バグ修正 `再来週`**: 再来週も同じ問題を内包している。

---

### 差分3: 朝昼夕夜の扱い

| | renderer | GAS |
|---|---|---|
| 入力 | `朝 ミーティング` | ← |
| startTime | `null`（時刻なし・全日イベント） | `08:00`（朝→8時変換） |
| 入力 | `夕方 商談` | ← |
| startTime | `null` | `17:00` |

**原因**: GAS は前処理で `朝→8時`、`昼→12時`、`夕方→17時`、`夜→19時` に文字列置換してから時刻パースする。renderer は `夕方` の後ろに数字がない場合は時刻として扱わない。  
**どちらが正しいか**: 文脈依存。「夕方 商談」= 時刻なし・全日（renderer が正しい）vs 夕方=17時（GAS はユーザー意図に近い場合もある）。  
**SPEC の2026-05-06 バグ修正 `朝昼夕方夜変換`**: この差分がまさにそのバグ修正に該当する。renderer で修正済み、GAS には旧ロジックが残っている。

### 差分4: endTime 自動付与

| | renderer | GAS |
|---|---|---|
| 入力 | `明日14時 打ち合わせ` | ← |
| startTime | `14:00` | `14:00`（一致） |
| endTime | `15:00`（+1h 自動付与） | `null` |

**原因**: renderer は単独時刻の場合に endTime = startTime + 1h を自動付与する。GAS は明示的な「〜」表記がない場合は endTime を null にする。  
**どちらが正しいか**: どちらも設計選択。Google カレンダー登録時は GAS 側でデフォルト1h を使うため実用上問題なし。Phase 5 後半で一本化時に方針統一が必要。  
**SPEC の2026-05-06 バグ修正 `〜から〜まで`**: 明示的な範囲指定に関するバグ（別ケース）。

---

### 差分5: `N/M` スラッシュ日付の挙動差

| | renderer | GAS |
|---|---|---|
| 入力 | `4/10 14:00 営業` | ← |
| date | `2026-04-10` | `2026-04-08`（今日に化ける） |

**原因**: GAS の時刻正規表現 `/(\d{1,2})[:：時](\d{0,2})/` が `4/10` の `0` 部分と `14:00` をマッチさせる前に、`/N月M日/` パターンにヒットしない。renderer は `/` をスラッシュ日付として解釈する専用パターンを持つ。  
**どちらが正しいか**: renderer が正しい。GAS は `4/10` 形式を認識できない。
